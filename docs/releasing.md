# Releasing `jarify-jetbrains`

This repo mirrors `../jarify` release flow: automated release PR prep, then automated publish on merge. JetBrains Marketplace still requires one manual bootstrap upload before the publish workflow can succeed.

## Human release flow

1. Land feature/fix/docs changes on `main` with conventional-commit PR titles.
2. `prepare-release.yml` computes next semver from unreleased conventional commits, bumps `pluginVersion` in `gradle.properties`, regenerates `CHANGELOG.md`, and opens or updates `release/vX.Y.Z`.
3. Review release PR:
   - `CHANGELOG.md`
   - bumped `pluginVersion`
   - any release-blocking verifier failures
4. Merge release PR.
5. `publish.yml` runs on the `gradle.properties` change on `main` — normally from that merged release PR — then:
   - `mise run test`
   - `mise run build`
   - `mise run verify`
   - upload `build/distributions/*.zip`
   - `mise run publish`
   - create git tag `vX.Y.Z`
   - create GitHub Release with generated notes and plugin ZIP asset

No manual tagging needed.

## First Marketplace upload

JetBrains requires the first plugin publication to be uploaded manually before later CI-based publishes can use `publishPlugin`.

Before first upload:

1. Sign in to JetBrains Marketplace / Hub with publishing account.
2. Accept Marketplace developer agreement if prompted.
3. Create vendor profile if prompted.
4. Run local preflight:

   ```bash
   mise run test
   mise run build
   mise run verify
   ```

5. Upload the plugin ZIP manually in Marketplace:
   - local build: upload `build/distributions/jarify-jetbrains-<version>.zip`
   - GitHub Actions artifact: download `jarify-jetbrains-plugin`, unzip the
     artifact archive once, then upload the inner
     `jarify-jetbrains-<version>.zip`

Do not upload the GitHub artifact wrapper ZIP directly. Marketplace expects the
plugin ZIP itself, whose root contains `jarify-jetbrains/lib/*.jar`.

After the plugin exists in Marketplace, automated publish on `main` can succeed.

## Troubleshooting publish failures

If `publish.yml` fails during `mise run publish` / `./gradlew publishPlugin`
with `Cannot find plugin`, JetBrains Marketplace does not have the initial
listing yet. CI can update an existing listing, but it cannot create the first
Marketplace listing for `com.amfaro.jarify`.

Recovery:

1. Run local preflight:

   ```bash
   mise run test
   mise run build
   mise run verify
   ```

2. Manually upload the plugin ZIP in JetBrains Marketplace to create the
   `com.amfaro.jarify` listing. If using the GitHub Actions artifact, unzip the
   downloaded artifact first and upload the inner
   `jarify-jetbrains-<version>.zip`, not the artifact wrapper ZIP.
3. Confirm `JETBRAINS_MARKETPLACE_TOKEN` is configured in repository secrets and
   belongs to the account that owns the listing.
4. Rerun the `Publish` workflow with `dry_run=false`, or rerun the failed job.

## Versioning and release notes

- `pluginVersion` in `gradle.properties` is release source of truth.
- `prepare-release.yml` updates it automatically from conventional commits.
- `CHANGELOG.md` is generated by `git-cliff` and reviewed in the release PR.
- Marketplace/GitHub release notes come from generated changelog plus GitHub release notes.
- Marketplace rejects duplicate plugin versions.

## Local preflight

Use same checks as CI before manual publish or before debugging release failures:

```bash
mise run test
mise run build
mise run verify
```

Artifacts land in `build/distributions/*.zip`.

## Secrets

Release automation uses GitHub Actions-native credentials only. No Doppler or GitHub App is required for this repo.

### Automatic

| Secret | Where used | Notes |
|---|---|---|
| `GITHUB_TOKEN` | `prepare-release.yml`, `publish.yml` | provided automatically by GitHub Actions; used for checkout, branch push, PR creation, and GitHub Release |

### Required repository secrets

| Secret | Where used | Notes |
|---|---|---|
| `JETBRAINS_MARKETPLACE_TOKEN` | `publish.yml` | Marketplace personal access token used by `publishPlugin` |

### Optional signing secrets

| Secret | Where used | Notes |
|---|---|---|
| `PRIVATE_KEY` | `publish.yml` / Gradle signing | plugin signing private key |
| `PRIVATE_KEY_PASSWORD` | `publish.yml` / Gradle signing | password for private key |
| `CERTIFICATE_CHAIN` | `publish.yml` / Gradle signing | certificate chain for signed release |

### JetBrains Marketplace token setup

1. Sign in to JetBrains Marketplace / Hub with publishing account.
2. Open `My Tokens`.
3. Create personal access token for plugin publishing.
4. Store it in this repo under **Settings → Secrets and variables → Actions** as `JETBRAINS_MARKETPLACE_TOKEN`.

Never commit or print token/signing values.

## Workflow files

- `.github/workflows/ci.yml` — PR/main validation, including `mise run verify`
- `.github/workflows/prepare-release.yml` — create or update release PR
- `.github/workflows/publish.yml` — publish on `main` when `gradle.properties` changes

## Important caveats

- Squash-merge PR titles must follow conventional commits or `git-cliff` will skip them for version bumping.
- Marketplace review still applies after upload; publish success does not guarantee immediate public availability.
- Signing is wired and recommended, but kept optional so local unsigned builds still work.
